# 二维码与失效兜底页专项设计

## 1. 目标

- 保证二维码长期可访问。
- 当短链失效、页面下线或数据异常时，用户仍可获得有效引导，不出现死链。

## 2. 二维码生成策略

- 二维码内容固定为短链：`https://q.yourdomain.com/q/{code}`。
- 不直接写展示页真实路径，不写临时签名 URL。
- 建议二维码生成后可重新下载，内容保持不变（同一个 `code`）。

## 3. 短链解析逻辑（Worker）

伪代码：

```text
input: code
1) 查询 share_links
2) 若不存在 -> redirect(fallback)
3) 若 status != active -> redirect(fallback)
4) 若 expire_at 不为空且当前时间 > expire_at -> 标记 expired 并 redirect(fallback)
5) 查询页面 publish_status
6) 若页面不存在或非 published -> redirect(fallback)
7) 记录 scan 事件
8) redirect(/s/{slug})
```

## 4. 兜底页触发条件

- `code` 不存在
- 短链状态为 `disabled/expired`
- 页面状态不是 `published`
- 数据读取异常（D1/KV临时异常）
- 目标页面被删除或未找到

## 5. 兜底页内容规范

兜底页建议包含：

- 友好提示：该链接已失效或内容已更新
- 企业联系方式：电话、微信二维码、地址（可配置）
- 操作引导：
  - 返回官网首页
  - 联系客服
  - 重新获取最新二维码（可选）

## 6. 兜底页路由建议

- 固定路由：`https://www.yourdomain.com/fallback`
- 支持附带参数：
  - `?code=AB12CD`
  - `?reason=expired|disabled|notfound|error`

示例：

`https://www.yourdomain.com/fallback?code=AB12CD&reason=expired`

## 7. 兜底 URL 优先级

1. `share_links.fallback_url`（短链级自定义）
2. 系统默认兜底页（全局配置）
3. 官网首页（最终兜底）

## 8. 监控与告警

- 指标：
  - 每日兜底触发次数
  - 触发原因分布
  - 失效短链 TOP10
- 告警建议：
  - 1 小时内 `reason=error` 超阈值触发告警
  - 某企业 `expired/disabled` 占比异常触发提醒

## 9. 运营建议

- 已投放线下物料的短码，不建议删除，只做状态变更。
- 页面改版优先复用原短码，避免重新印刷二维码。
- 兜底页显示企业统一客服电话，防止流量流失。
