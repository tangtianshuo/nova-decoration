# 测试计划与验收标准

## 1. 测试范围

- 管理端：登录、公司资料、素材上传、页面发布、二维码下载。
- 访客端：扫码访问、内容展示、视频播放、联系方式点击。
- 兜底链路：短链不存在/失效/页面下线/异常时跳转兜底页。

## 2. 功能测试用例（关键）

### 2.1 登录与权限

- 正确账号密码可登录。
- 错误密码返回统一错误提示。
- 未登录访问后台接口返回 401。

### 2.2 上传链路

- 获取上传签名成功。
- 直传 R2 成功后素材记录可见。
- 非法文件类型或超限大小被拒绝。

### 2.3 发布与短链

- 页面发布后可生成短链与二维码。
- 同页面重复发布可复用短链。
- 短链访问能正确跳转到展示页。

### 2.4 兜底页

- 不存在 code -> 跳转兜底页。
- 短链 `disabled/expired` -> 跳转兜底页。
- 页面 `offline` -> 跳转兜底页。
- 兜底页显示联系方式与回流入口。

## 3. 性能测试（建议）

- 扫码入口 `GET /q/:code` 在缓存命中场景下 p95 < 300ms。
- 展示页首屏加载（含首图）建议 p75 < 3s（4G 网络）。

## 4. 安全测试（建议）

- 登录防暴力破解（限流生效）。
- 上传签名不可复用/过期后失效。
- 越权访问企业数据时被拒绝。

## 5. 验收标准（Go-Live）

- 主链路“上传 -> 发布 -> 生成二维码 -> 扫码展示”全部通过。
- 兜底页触发逻辑全部通过。
- 无阻断级安全问题。
- 监控日志可追踪关键事件。
