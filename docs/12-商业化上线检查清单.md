# 商业化上线检查清单

## 1. 计费

- 套餐表 `plans` 已初始化并包含 `free/pro/enterprise`。
- 每个租户都有 `tenant_subscriptions` 记录。
- 支持套餐变更接口与交易流水记录（`billing_transactions`）。
- 生产环境配置 `BILLING_WEBHOOK_SECRET` 并完成回调签名校验联调。

## 2. 配额

- 配额读取接口 `/api/quotas/me` 可返回套餐限制与当前用量。
- 上传签名、上传完成、页面创建已接入配额校验。
- 扩容接口 `/api/quotas/overrides` 可生效并可追溯创建人和原因。

## 3. 权限

- `tenant_admin`：可执行写操作（公司资料、页面、素材、短链、套餐变更）。
- `tenant_editor`：可编辑业务内容（页面、素材、短链），不可改公司资料和套餐。
- `tenant_viewer`：只读访问，写操作应统一返回 403。
- `super_admin`：仅平台接口使用，不应绕过租户数据边界。

## 4. 发布治理

- CI 对 `dev/release/main` 分支执行阻断式 `lint/test/build`。
- `release` 分支触发质量闸门工作流，手动发布前必须通过。
- 发布前执行 `node scripts/api-smoke.mjs`，并保留输出记录。
- 变更需记录发布版本、提交哈希、执行人、上线时间与回滚策略。

## 5. 上线前验证（必做）

- 新租户注册后可自动获得默认订阅和基础配额。
- 套餐变更后配额即时生效。
- 配额超限时接口返回明确错误码与提示。
- 到期或欠费状态下，策略限制符合产品定义（只读/停写/停用）。
- 关键链路观测可用：登录、上传、发布、短链访问、支付回调。
