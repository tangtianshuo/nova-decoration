# 系统架构设计

## 1. 架构目标

- 无传统服务器运维负担
- 兼顾上传安全、二维码长期可访问、低成本
- 支持后续平滑扩展为多租户 SaaS

## 2. 总体架构

### 2.1 组件

- Web 前端（Vercel）
  - 管理端：企业资料、素材管理、展示页编辑、二维码下载
  - 访客端：扫码展示页
- API 网关（Cloudflare Workers）
  - 鉴权、上传签名、短链重定向、公开数据读取、埋点写入
- 对象存储（Cloudflare R2）
  - 图片、视频、可选二维码文件
- 数据存储（Cloudflare D1）
  - 公司、用户、页面、素材、短链、事件
- 缓存（Cloudflare KV，可选）
  - 短链映射缓存、热点页面缓存

### 2.2 请求流

1. 管理端调用 Worker 获取上传签名。
2. 浏览器直传文件到 R2。
3. 上传完成后回调 Worker 写入素材记录。
4. 页面发布后 Worker 生成短码。
5. 访客扫码访问 `q` 短链，Worker 记录扫码并重定向到展示页。

## 3. 关键设计决策

- 采用“前端直传 R2 + Worker 授权签名”，避免前端持有长期密钥。
- 二维码内容统一使用短链，不直接绑定资源 URL。
- 短链状态支持 `active`、`disabled`、`expired`，非 active 进入兜底页。
- 公开展示数据由 Worker 聚合输出，避免前端直接暴露内部结构。

## 4. 可用性与持久化

- 短链数据存 D1，定时备份。
- R2 禁止误删策略（生产桶启用版本管理或操作审计策略）。
- 兜底页固定可用地址（例如 `https://www.xxx.com/fallback`）。

## 5. 可扩展点

- 增加队列（Cloudflare Queues）处理异步任务：
  - 视频转码
  - 封面抽帧
  - 异步审核
- 接入 Cloudflare Analytics Engine 或第三方 BI。

## 6. 容灾与降级

- D1 短时异常：短链优先查 KV 缓存，读取失败返回兜底页。
- Bilibili 不可播放：降级显示封面 + 跳转外链按钮。
- 媒体资源 404：展示占位图并记录错误事件。
