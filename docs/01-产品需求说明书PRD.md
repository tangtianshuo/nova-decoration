# 产品需求说明书（PRD）

## 1. 背景与目标

装修企业需要快速对外展示案例与产品，并通过二维码在门店、工地、宣传单页中传播。本项目提供一个轻量化系统，让企业上传素材后即可生成可长期访问的二维码页面。

## 2. 目标用户

- 企业管理员：维护公司资料、上传素材、配置展示内容、生成二维码。
- 访客用户：通过扫码查看公司介绍、产品介绍、案例图/视频并联系企业。

## 3. 核心场景

1. 企业创建展示页并上传图片/视频。
2. 企业录入 Bilibili 视频链接作为展示素材。
3. 企业下载二维码用于线下物料投放。
4. 访客扫码查看内容并通过电话/微信联系企业。
5. 短链失效时自动跳转兜底页，避免死链。

## 4. 功能范围（MVP）

### 4.1 企业后台

- 注册/登录（首期可仅管理员创建账号）。
- 公司资料管理：logo、名称、简介、联系方式、地址。
- 展示页管理：创建、编辑、发布、下线。
- 素材管理：
  - 图片上传（R2）
  - 视频上传（R2）
  - Bilibili 链接录入
- 二维码生成与下载（PNG/SVG 任选其一作为首期）。

### 4.2 访客侧

- 扫码进入公开展示页。
- 展示公司介绍、产品介绍、联系方式。
- 展示图片轮播、视频播放、Bilibili 外链内容。
- 页面加载失败或内容下线时，进入兜底页。

### 4.3 统计（基础版）

- 扫码次数（短链层）
- 展示页访问次数
- 视频点击/播放次数（前端埋点）

## 5. 非功能需求

- 可用性：扫码访问可用率 >= 99.9%（月度）
- 性能：国内主流网络首屏加载建议 <= 3s（缓存命中场景）
- 安全：上传权限控制、鉴权访问、接口限流
- 可维护：关键日志可追溯，支持快速定位短链故障

## 6. 业务规则

- 二维码内容必须是平台短链，不允许直接写入临时签名 URL。
- 已发布页面修改后可重新发布，原短码默认不变。
- 页面下线后，短码不直接 404，而是跳转兜底页。
- Bilibili 外链不可播放时，展示封面与“前往 Bilibili 查看”按钮。

## 7. 关键指标（首期）

- 企业建页成功率 >= 95%
- 二维码生成成功率 >= 99%
- 扫码后到达可展示页面成功率 >= 99%
- 兜底页触发率（异常链路）可监控可追踪

## 8. 风险与约束

- 第三方视频平台外链策略可能变更。
- 纯前端不具备安全上传能力，因此必须通过 Worker 颁发签名上传权限。
- 若域名变更，需保障旧短链可迁移或做 301 兼容。

## 9. 版本规划

- v1.0：单企业账号、基础展示页、二维码与兜底页、基础统计。
- v1.1：多账号角色、模板化展示页、批量导入素材。
- v1.2：线索收集、数据看板、营销追踪参数。
