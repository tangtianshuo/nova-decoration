# 安全与合规设计

## 1. 认证与授权

- 后台接口全部要求 JWT。
- JWT 中必须包含 `user_id`、`company_id`、`role`。
- 所有写操作校验 `company_id` 一致性，禁止越权操作。

## 2. 上传安全

- 仅通过 Worker 签发短时上传 URL（有效期建议 5 分钟）。
- 限制上传文件类型与大小：
  - 图片：`jpg/jpeg/png/webp`，建议 <= 10MB
  - 视频：`mp4/mov`，建议 <= 200MB（首期）
- 上传路径按企业隔离：`company/{company_id}/...`
- 禁止前端持有 R2 长期密钥。

## 3. 接口安全

- 登录接口：IP + 账号维度限流。
- 上传签名接口：按企业配额限流。
- 公开短链接口：高并发保护与基础防刷。
- 关键操作记录审计日志（创建/发布/下线/短链状态修改）。

## 4. 数据安全

- 密码存储必须使用强哈希（Argon2 或 bcrypt）。
- D1 定期导出备份（每日）。
- 事件日志按周期归档，减少主库压力。

## 5. 内容合规

- 提供页面下线能力（违规可快速拦截）。
- Bilibili 外链遵循第三方平台规则，不绕过平台限制。
- 展示页需包含企业真实联系方式与免责声明（可选）。

## 6. 隐私与合规建议

- 后台提供《隐私政策》《服务协议》入口。
- 仅采集必要数据（扫码、访问、播放等匿名统计）。
- 对包含手机号等敏感字段进行最小可见原则控制。
