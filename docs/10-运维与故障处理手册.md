# 运维与故障处理手册

## 1. 日常巡检

- 检查 Worker 请求量、错误率、平均响应时间。
- 检查短链访问成功率与兜底页触发率。
- 检查 R2 存储增长与异常流量。
- 检查 D1 可用性与慢查询。

## 2. 常见故障与处理

### 2.1 扫码后全部进入兜底页

排查步骤：

1. 检查 `q.yourdomain.com` DNS 与 Worker 路由是否正常。
2. 检查 D1 查询是否异常，确认 `share_links` 是否可读。
3. 检查 `DEFAULT_FALLBACK_URL` 是否被误配置。

### 2.2 上传失败

排查步骤：

1. 检查签名 URL 是否过期。
2. 检查 R2 CORS 是否允许来源域名。
3. 检查文件类型/大小是否超过限制。

### 2.3 Bilibili 无法播放

排查步骤：

1. 校验链接格式与 BV 号。
2. 判断是否触发平台策略限制。
3. 降级展示封面 + 跳转按钮。

## 3. 应急预案

- Worker 异常时，短链统一 302 到官网或全局兜底页。
- D1 异常时，优先读取 KV 缓存映射（若已启用）。
- 发布高风险变更前，预留回滚版本并设置变更窗口。

## 4. 备份与恢复

- D1 每日备份，保留 30 天。
- 关键配置（环境变量、域名解析）导出备案。
- 演练恢复流程：从备份恢复短链与页面核心表。

## 5. 运营看板建议

- 总扫码量、独立扫码人数（可选）
- 兜底页触发次数与原因占比
- 企业维度页面访问排行
- 热门视频/图片点击排行
