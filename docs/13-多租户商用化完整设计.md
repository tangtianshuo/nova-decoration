# 多租户商用化完整设计

## 1. 目标定义

- 从“可用”升级到“可收费、可运营、可审计、可发布治理”。
- 支持租户自助订阅、容量扩容、超管运营管理、关键经营指标可观测。
- 在欠费/取消等状态下，系统行为可预测（只读或停写），防止越权与失控成本。

## 2. 领域模型

### 2.1 计费域

- `plans`：套餐定义（价格、配额上限、状态）。
- `tenant_subscriptions`：租户订阅状态机（trialing/active/past_due/canceled）。
- `billing_transactions`：交易流水（事件类型、金额、渠道、状态、原始载荷）。

### 2.2 配额域

- `tenant_quota_overrides`：人工扩容（额外空间/资产/页面/成员）。
- `usage_monthly_snapshots`：按月累计上传流量等计量。

### 2.3 治理域

- `release_audits`：发布审计记录（版本、提交、执行人、质量门禁）。

## 3. 关键策略

- 写操作门禁：仅 `active/trialing` 订阅可写。
- RBAC：`tenant_admin`、`tenant_editor`、`tenant_viewer` 严格分层。
- 配额校验前置：上传签名、上传完成、页面创建均校验配额。
- 支付回调幂等：`provider_txn_id` 重复回调直接去重返回成功。
- 风险控制：登录/上传/回调路径设置基础限流，抑制暴力流量。

## 4. API 设计分层

- 租户自助：`/api/billing/*`、`/api/quotas/*`
- 超管运维：`/api/platform/tenants/:tenantId/commercial/*`、`/api/ops/metrics/overview`
- 业务域：素材/页面/短链/公司写接口统一接入 RBAC + 商业状态校验

## 5. 前端设计

- 租户后台：
  - 计费中心：当前订阅、套餐切换、扩容购买、交易记录。
  - 配额中心：限制值、实时用量、剩余量与进度条。
- 超管后台：
  - 商业运营：租户套餐切换、扩容包发放、MRR 与风险指标。
  - 租户管理页增加商业运营跳转入口。

## 6. 发布治理设计

- `CI`：对 `dev/release/main` 强制执行 `lint/test/build`。
- `Release Governance`：
  - `release` 分支 push 自动跑质量闸门。
  - 手动 `workflow_dispatch` 执行 Worker 生产发布。
- 发布前必跑：`node scripts/api-smoke.mjs`

## 7. 验收标准

- 新租户自动具备默认套餐与配额。
- 欠费租户无法执行写操作，读取仍可用。
- 超配额请求返回明确错误码与提示文案。
- 超管可在线调整套餐/扩容并即时生效。
- 平台可查看 MRR、欠费数、存储 Top 租户。
