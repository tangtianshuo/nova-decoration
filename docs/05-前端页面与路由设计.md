# 前端页面与路由设计（Next.js）

## 1. 管理端路由

- `/login`：登录页
- `/dashboard`：概览（页面数、扫码数、最近发布）
- `/company/profile`：公司资料
- `/assets`：素材列表
- `/assets/upload`：素材上传
- `/pages`：展示页列表
- `/pages/new`：新建展示页
- `/pages/[id]/edit`：编辑展示页
- `/pages/[id]/share`：二维码与分享设置

## 2. 访客端路由

- `/s/[slug]`：展示页主入口
- `/fallback`：兜底页（短链失效、页面下线、异常场景）

## 3. 页面模块模型（建议）

展示页由 `blocks` 组成，按 `sort_order` 渲染：

- `hero`：封面图 + 标题 + 简介
- `company_intro`：公司介绍
- `product_intro`：产品介绍
- `gallery`：图片轮播
- `video`：上传视频或 Bilibili 外链
- `contact`：电话、微信、地址、地图链接

## 4. 关键交互

- 保存草稿：仅保存，不更新短链目标版本。
- 发布页面：生成/复用短链，并更新访客可见版本。
- 下线页面：短链自动进入兜底页。
- 二维码下载：支持 PNG（首期），带默认品牌标识（可选）。

## 5. 前端状态管理建议

- 轻量方案：React Query + Zustand
- 统一处理 API 错误码，401 自动跳登录
- 上传过程显示进度、失败重试、成功回写

## 6. 可访问性与兼容性

- 移动端优先（扫码主要来自手机）
- 微信内置浏览器兼容（视频自动播放策略提示）
- 图片懒加载与响应式尺寸，优化首屏速度

## 7. SEO 与分享

- 公开页设置 `title/description/og:image`
- 兜底页文案明确，提供“返回官网/联系客服”入口
