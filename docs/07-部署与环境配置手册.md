# 部署与环境配置手册

## 1. 资源准备

- Cloudflare 账号（含 Workers、R2、D1、KV）
- Vercel 账号
- 自有域名（建议主域名与短链域名分开）

建议域名规划：

- `www.yourdomain.com`：前端站点
- `api.yourdomain.com`：Worker API
- `q.yourdomain.com`：短链入口
- `media.yourdomain.com`：R2 媒体域名

## 2. Cloudflare 配置

### 2.1 R2

- 创建 bucket：`nova-media-prod`
- 绑定自定义域名：`media.yourdomain.com`
- 配置 CORS（允许前端域名 PUT/GET）

### 2.2 D1

- 创建数据库：`nova-prod`
- 执行 `03-数据模型与DDL草案.md` 中 DDL

### 2.3 KV（可选）

- 创建 namespace：`SHORT_LINK_CACHE`
- 用于短链热点缓存

### 2.4 Worker

- 绑定资源：R2、D1、KV
- 设置环境变量：
  - `JWT_SECRET`
  - `DEFAULT_FALLBACK_URL`
  - `APP_WEB_URL`
  - `QRCODE_BASE_URL`

## 3. Vercel 配置

- 关联 Git 仓库并部署前端。
- 配置环境变量：
  - `NEXT_PUBLIC_API_BASE=https://api.yourdomain.com`
  - `NEXT_PUBLIC_WEB_BASE=https://www.yourdomain.com`
- 生产分支保护：仅主分支触发正式发布。

## 4. DNS 与路由

- Cloudflare DNS 中添加 CNAME/A 记录指向 Vercel 与 Worker。
- `q.yourdomain.com` 推荐仅用于短链解析，减少规则冲突。

## 5. 安全策略（上线前）

- 开启 WAF 托管规则。
- 登录、上传、事件上报接口启用限流。
- 管理端登录接入 Turnstile。
- 强制 HTTPS 与 HSTS。

## 6. 发布流程建议

1. 在测试环境验证：
   - 登录
   - 上传图片/视频
   - 创建页面并发布
   - 二维码扫码与兜底跳转
2. 生产环境灰度一批企业账号。
3. 观察 24 小时指标后全量放开。
